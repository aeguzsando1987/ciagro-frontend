feat(visor-datos): modos de mapa Satélite/Híbrido sin API key

Sesión 20 (rama dev). Reimplementación correcta del selector de modos
que en la sesión 19 quedó como spike revertido.

Causa raíz del fallo previo
- VITE_MAPTILER_KEY era un placeholder → el híbrido MapTiler daba 403
  y "no se veía ningún cambio". setTiles reactivo además era frágil.

Solución (sin API key, robusta)
- ESRI_STYLE incorpora las capas de referencia GRATUITAS de ESRI
  (World_Transportation + World_Boundaries_and_Places, PNG transparente)
  con visibility:'none' inicial, declaradas en el estilo base → siempre
  al fondo, las capas de datos quedan encima.
- Hook compartido useMapMode(mapRef): el modo híbrido solo alterna la
  visibilidad de esas capas vía setLayoutProperty (espera al evento
  'load' si el estilo no cargó). Cero remount, cero reordenamiento.
- Eliminado MapTiler / BLANK_STYLE / MAP_MODES.tiles de mapModes.ts.

Alcance por mapa
- Navegación (ProducerRanchesMap, RanchPlotsMap): selector Satélite/Híbrido.
- Vista de capas de datos (AspersionMap, heatmap): selector retirado a
  pedido; queda fijo en satélite (carreteras/etiquetas estorban el heatmap).

Verificación: tsc --noEmit 0 errores; 177/177 vitest verdes; modo
confirmado manualmente por el usuario.
